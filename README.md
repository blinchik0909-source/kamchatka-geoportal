# Геопортал «Геологические достопримечательности Камчатки»

Статический геопортал на **MapLibre GL JS** (+ **Turf.js** для измерений) без
серверной части и без сборщика. Работает как набор статичных файлов — его можно
бесплатно разместить на **GitHub Pages** или **Cloudflare Pages**.

## Возможности
- Управление ГИС-слоями (вкл/выкл слоёв) — вкладка «Объекты».
- Картографическая основа из OpenStreetMap — вкладка «Карт. основа»: отдельные
  включаемые слои (вода, растительность, дороги, ж/д, здания, ООПТ, админграницы,
  населённые пункты, POI и др.), подгружаются на лету через публичный Overpass API
  по видимой области карты.
- Открытые подложки: OpenStreetMap, рельеф (OpenTopoMap), спутник (ESRI)
  (кнопка переключения в правом верхнем углу карты).
- Простые измерения расстояний и площадей (кнопка «линейка» в правом верхнем углу).
- Поиск объектов по названию.
- При клике по объекту — фото, текстовое описание и таблица атрибутов.

### О слоях OpenStreetMap (вкладка «Карт. основа»)
- Данные берутся напрямую из OSM через Overpass API (бэкенд не нужен).
- Слой грузится только при достаточном приближении (у каждого слоя свой `minZoom`)
  и только по текущей области карты — иначе запрос был бы слишком большим.
- При перемещении/зуме включённые слои автоматически перезапрашиваются.
- Состав слоёв, цвета, `minZoom` и Overpass-запросы настраиваются в `js/config.js`
  (массив `osmLayers`).
- Overpass — публичный бесплатный сервис с ограничением по частоте запросов;
  для интенсивного использования можно поднять свой инстанс или сменить зеркало
  (`overpassEndpoint` в `js/config.js`).

## Структура
```
kamchatka-geoportal/
├── index.html          # разметка страницы
├── css/style.css       # стили
├── js/config.js        # КОНФИГ: подложки, слои, поля попапа
├── js/app.js           # движок карты (менять не нужно)
├── data/               # ваши GeoJSON-слои
│   ├── volcanoes.geojson
│   └── springs.geojson
└── images/             # фотографии объектов (необязательно)
```

## Как подключить свои данные
1. Положите свои файлы `*.geojson` в папку `data/`.
2. В каждом объекте (`properties`) задайте поля (имена настраиваются в `js/config.js`):
   - `name` — название (заголовок попапа и поиск);
   - `type` — тип объекта (бейдж);
   - `description` — текстовое описание;
   - `photo` — путь к фото (`images/avacha.jpg`) или URL;
   - любые другие поля попадут в таблицу атрибутов.
3. Опишите слои в `js/config.js` → массив `layers` (id, name, путь к geojson, цвет).
4. Фото положите в `images/` (рекомендуется сжать до ширины ~1000–1200 px).

## Запуск локально
Из-за `fetch()` файлы нужно открывать через локальный сервер (а не двойным кликом):
```bash
# Python 3
python -m http.server 8000
# затем открыть http://localhost:8000
```

## Бесплатный хостинг

### Вариант A — GitHub Pages
1. Создайте репозиторий на GitHub и загрузите содержимое этой папки.
2. Settings → Pages → Source: ветка `main`, папка `/ (root)` → Save.
3. Через минуту сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.

### Вариант B — Cloudflare Pages (рекомендуется: больше трафика, без «засыпания»)
1. Залейте код в репозиторий GitHub.
2. Cloudflare Dashboard → Workers & Pages → Create → Pages → Connect to Git.
3. Build command — пусто; Build output directory — `/` (корень).
4. Deploy. Получите адрес `https://<project>.pages.dev`.

## Создание нового геопортала на этом движке
Скопируйте папку, замените `data/*.geojson`, обновите `js/config.js`
(центр карты, слои, подложки). Код в `js/app.js` менять не требуется.

## Примечания по подложкам
- OSM-тайлы подходят для прототипа/небольшого трафика; при росте используйте
  провайдера с API-ключом (MapTiler, Stadia, Carto).
- Спутник ESRI и OpenTopoMap требуют указания атрибуции (она уже добавлена).
